# 합성 원칙 — 하네스 컴포넌트를 섞으면 언제 좋아지고 언제 나빠지나

> 질문: 하네스에 컴포넌트(에이전트·컨텍스트·게이트·패턴)를 *더* 넣으면 성능이 오르나 내리나?
> 근거: 우리 12실험(측정) + 검증한 동료심사 논문(`docs/papers/`) + 외부 하네스 비교([EXTERNAL_HARNESS_COMPARISON.md](EXTERNAL_HARNESS_COMPARISON.md)).
> 핵심 발견: **우리 12실험은 사실 "검증 컴포넌트를 더했을 때의 합성 곡선"을 잰 것이었다.** 합성은 *공짜가 아니고, 무료도 아니다* — 조건에 달렸다.

> ⚠️ **"법칙"이라는 단어 주의**: 아래 "독립성 법칙·비대칭 법칙"은 **2~3개 데이터포인트(대부분 천장·동어반복·n=1~2)**에서 *사후 해석*한 것이다. 메커니즘 서사로는 그럴듯하나 *측정으로 분리·확증된 법칙*은 아니다 — "법칙"은 *작업가설(working hypothesis)*로 읽어라(증거 한계: [eval RESULTS §신뢰도](../../../scripts/harness/eval/RESULTS.md)).

## 0. 한 줄 명제

> 합성은 **섞는 행위 자체**로 나빠지지 않는다. **겹치거나(redundant)·충돌하거나(interfere)·희석하거나(dilute)·과제약하거나(saturate)·과부하(overload)** 될 때 나빠진다. 마진 컴포넌트가 **직교적이고, 새 실패모드를 덮고, 조건부로 발동하고, 진짜 목표를 재면** — 더해도 (중립~)이득이다.

## 1. 성능을 떨어뜨리는 5축 (각각 메커니즘·증거·완화)

| 축 | 메커니즘 | 무엇이 망가지나 | 우리 증거 | 완화 |
|----|---------|----------------|-----------|------|
| **A. 간섭(Interference)** | 공유 가변자원 + 동시 writer + 무조율 → 충돌 | *정확성*(틀린 결과) | 멀티에이전트 코딩 충돌(Cognition) | 파일 disjoint + 단일 통합자 |
| **B. 희석(Dilution)** | 유한한 attention, 무관 컨텍스트↑ → S/N↓ | *검색/주의*(중간 소실) | lost-in-the-middle(Liu 2307.03172); H8 fixture 주석이 정답 leak·오도 | plan-search 절추출(96%↓), 조건부 주입 |
| **C. 중복(Redundancy)** | 과제가 이미 풀림 → 추가 체크 무의미 | *비용/지연*(정확도 아님) | H2(렌즈 1vs3 천장), 11실험 ceiling | self-exclude(none+low), 역할분담 |
| **D. 과제약(Saturation)** | 게이트 압력 → 목표 대체(목표 대신 게이트 최적화) | *생성자*(reward-hack) | assertion 약화·테스트 삭제(2605.01471); 우리 가드7 | 진짜 목표 측정, soft warn, reward-hack 감시 |
| **E. 과부하(Overload)** | 공유 rate-limit 자원 + 버스트 | *처리량*(529) | 워크플로우 동시 다수 → 529 | 직렬화·동시성 cap·backpressure |

→ 5축은 *서로 다른 것*을 망가뜨린다. "섞으면 느려진다"는 단일 현상이 아니라 **5개의 별개 실패모드**다.

## 2. ★ 핵심 통찰 — 중복 컴포넌트의 가치 ∝ *오차 독립성*

self-consistency(Wang ICLR'23)는 다수 샘플 다수결이 낫다고 하지만, **전제는 "오차가 독립"**이다. 우리 H2에서 3렌즈가 *만장일치*였던 건 — 결정론적 코드에 대한 capable 모델의 판단이 **거의 결정론적(저오차·상관)** 이라서다. **상관된 오차는 다수결로 안 줄어든다** → 렌즈를 더해도 정확도 0 이득(천장).

반대로 v-next에서 grounding이 이긴 건, DB 쿼리가 **모델 추론과 *독립된* 정보원**이라 오차 상관을 깨뜨렸기 때문이다.

> **통합 법칙**: 컴포넌트를 더해 얻는 이득은 그 컴포넌트의 오차가 기존 것과 **얼마나 독립인가**에 비례한다.
> - 같은 모델·같은 정보원(렌즈 복제) → 오차 상관 → 이득 0 (H2·11실험 ceiling)
> - 다른 정보원(실행·DB·외부도구) → 오차 독립 → 이득 큼 (v-next)
>
> 이게 우리 capstone("grounding은 추론으로 못 푸는 곳에만 정확도 이득")의 *더 깊은 이유*다 — grounding이 정확도를 사는 건 그게 **독립 정보원**일 때뿐.

## 3. ★ 핵심 통찰 — 합성의 수익은 *생성 vs 검증*에서 비대칭

- 외부 하네스(revfactory): 사전구성(스킬·역할·컨텍스트)을 **생성자에 더해** +60%(어려운 과제일수록↑). = *생성* 측 합성은 이득.
- 우리 12실험: 렌즈·grounding을 **검증자에 더해** 천장. = *검증* 측 합성은 포화(독립 정보원 빼고).

> **비대칭 법칙**: 합성을 *어느 쪽*에 하느냐로 수익이 갈린다.
> - **생성자 측**(scaffolding·design-first·컨텍스트 구조화) → 이득, 어려운 과제일수록 큼 (불확실성을 *사전에* 줄임)
> - **검증자 측**(렌즈·grounding 복제) → 포화, 독립 정보원일 때만 이득 (사후 확인은 이미 맞은 걸 재확인)
>
> 함의: 품질을 *올리려면* 검증자를 늘리기보다 **생성자에 구조를 더하라.** (우리가 아직 *측정 안 한* 축 = eval B.)

## 4. 합성 게이트 — 컴포넌트를 추가하기 전 5문답

새 하네스 조각(훅·렌즈·에이전트·패턴)을 넣기 전 체크:

1. **새 실패모드를 덮나?** (아니면 = 중복축 C → 비용만)
2. **오차가 기존과 독립인가?** (같은 모델·정보원이면 다수결 이득 0 → §2)
3. **공유 가변자원을 건드리나?** (예 → 간섭축 A, 조율 없으면 금지)
4. **무관할 때 조용한가?**(조건부 발동/self-exclude) (아니면 = 희석축 B·노이즈)
5. **진짜 목표를 재나, 게임 가능한 proxy인가?** (proxy면 과제약축 D → reward-hack)
6. (운영) **동시성 부담은?** (과하면 축 E → cap/직렬)

→ 5/6개가 안전하면 "다 추가"해도 안 떨어진다. 하나라도 걸리면 그 축의 완화를 먼저.

## 5. 우리 하네스가 이미 따르는 것 / 배울 것

**이미 따름** (그래서 섞여도 안 떨어짐):
- 훅 시점 분리(A·B 회피), self-exclude(C), 역할분담(C), 파일 disjoint(A), soft warn(D), 직렬화(E).

**이 분석에서 새로 배운 것**:
1. **우리 12실험 = 검증 측 합성 곡선**이었다 — "천장"은 *중복축*의 발현이고, v-next 신호는 *독립 정보원*의 발현. capstone을 합성 이론으로 재해석.
2. **품질 레버는 검증자 증식이 아니라 생성자 구조화** (§3 비대칭) → **eval B(생성 품질 A/B)가 진짜 다음 측정**. 우리가 검증만 재고 생성을 안 잰 게 사각지대.
3. **합성 게이트(§4)를 설계 규칙으로** — 앞으로 A(감독자)·C(generator) 추가 시 이 6문답을 통과시킨다. (A: 새 실패모드=가변배치✓, 독립✓, 공유자원=파일disjoint로 회피, 조건부✓, proxy아님✓, 동시성=cap → 통과.)
4. **독립성이 핵심 자원** — 검증 다양성을 *렌즈 수*가 아니라 *정보원 다양성*(실행·DB·외부문서)으로 설계해야 (그래서 6렌즈가 도구별로 다른 정보원을 쥐게 한 게 맞았다).

## 6. 한 줄

> 섞어서 나빠지는 건 5축(간섭·희석·중복·과제약·과부하) 중 하나에 걸릴 때뿐. **마진 컴포넌트의 가치 = 오차 독립성 × 새 실패모드 커버리지**, 그리고 그 수익은 **검증자보다 생성자 쪽에서 크다**. 합성 게이트 6문답을 통과시키면 "필요한 것 다 추가"는 안전하다.
