# 외부 하네스 비교 — revfactory/harness에서 배운 것

> 2026-06-07, `https://github.com/revfactory/harness` (README_KO.md + agent-design-patterns.md + claude-code-harness A/B) 심층 분석.
> 목적: 우리 하네스에 *진짜 필요한* 것만 분별해 흡수(따라하기 아님).

## 정체 — 다른 *종류*의 하네스

| | revfactory/harness | 우리 (AnchorIQ) |
|---|---|---|
| 본질 | **에이전트 팀을 *생성*하는 메타-팩토리**("하네스 구성해줘"→`.claude/agents`+`skills` 자동생성) | **한 코드베이스를 스펙에 붙들어두는 *엔포서***(훅·적대자·DLT) |
| 초점 | 도메인별 팀 아키텍처 *생성*(넓이·재사용) | 코드 *정합성·거버넌스*(깊이·측정) |
| 측정 | 생성 품질 +60%(n=15) | 검증 정확도(12실험, 경계 발견) |
| 멀티에이전트 | 에이전트 팀 + **SendMessage 실시간 상호수정** | 코딩은 **메시징 회피**(Cognition), 공유계약 fan-out |

→ 경쟁이 아니라 다른 레이어. 그들=L3 "팀 아키텍처 팩토리", 우리=프로젝트 거버넌스 엔포서.

## 신뢰도 검증 (그들의 핵심 주장)

**+60%(49.5→79.3, n=15) 주장은 방법론이 약하다**:
- baseline="프롬프트만" vs "`.claude/` 사전구성" — *생성* 품질 비교
- ❌ 채점자·블라인드 여부 불명, p-value/신뢰구간 없음, 과제별 점수 비공개, 한계 언급 0, **peer-review 아님**(Hwang 2026 출판예정·자가측정)
- 판정: "인상적이나 독립 검증 불가." 우리 eval이 더 엄밀(통제·반증가능·negative 정직보고·peer-reviewed 근거).
- 단 일치점: 그들도 **"어려운 과제일수록 개선↑"**(Basic +23.8%→Expert +36.2%) — 우리 capstone "경계=추론으로 못 푸는 곳"과 같은 방향.

## 6가지 팀 패턴 vs 우리

| 패턴 | 우리 보유? |
|------|-----------|
| 파이프라인(순차) | ◐ (PostToolUse 단계검증) |
| **팬아웃/팬인**(병렬→통합) | ✅ orchestrate-feature |
| 전문가 풀(라우팅) | ✅ 검증패널 6렌즈(차원별) |
| **생성-검증**(재시도 2~3회) | ✅ complete-stubs(가드6) — *그들과 독립적으로 같은 "max 2~3회" 결론* = 수렴 검증 |
| **감독자**(동적 부하분산, 공유 작업목록) | ❌ → 흡수 대상(A) |
| 계층적 위임 | ◐ (1단계 권장, 우리는 평탄) |

## 흡수 결정 (필요한 것만)

- **A 감독자 패턴** ✅ — 대규모/가변 독립작업(마이그레이션)에 정적 분해보다 나음. `parallel()`의 동시성 cap이 이미 *동적 디스패치*를 줌 → supervisor 워크플로우로 구현.
- **B 생성품질 A/B** ✅ — 우리 12실험은 전부 *검증* 정확도. **하네스가 *코드 생성*을 개선하는지** 미측정(그들의 +60% 축). 우리 eval 사각지대 → design-first 주입 on/off A/B.
- **C Track 2 generator** ✅(최종) — 우리가 미뤄둔 "하네스 재배포"의 성숙형이 그들의 *생성 스킬*. generator 스킬로 설계.
- **HEM 자기진화**(`/harness:evolve`) ⏭️ 스킵 — 초기↔출시 델타 학습. aspirational, 지금은 과조기.

## 안 따라할 것 (우리가 옳은 곳)

- **에이전트 간 메시징**: 그들은 실시간 상호수정이 품질 동력이라 봄. 우리는 *코딩에선* 안 함(공유 코드상태 충돌, Cognition). 그들 예시(소설·웹툰·마케팅)는 공유 코드상태가 없어 메시징 OK — **코드엔 우리 fan-out+계약이 맞다.**
- 그들은 프로젝트 정합성 *강제*가 없음(생성만). 우리 훅·적대자·DLT가 그 갭을 메운다.
