# ADVERSARY — PostToolUse 적대자의 recall / 오탐률 (검증 절반의 신뢰도)

> 사전등록: [`../dataset-adversary.json`](../dataset-adversary.json). 동기: DRIFT는 적대자가 버그 1개를 잡는 걸 봤을 뿐, *비율*은 미측정. 생성 절반은 GENQUALITY로 쟀으니 검증 절반의 'is it good'을 잰다.
> 설계: 4개 축 × (위반1+깨끗1) = 8 파일 "판별 쌍". 적대자가 쌍에서 위반만 막고 깨끗만 통과 → 진짜 판별. 블라인드(seeded/clean·무슨버그 모름), repo 파일 안 쓰고 프롬프트 inline.

### 점수
- **recall = 4/4 (100%)** — seeded 위반 전부 적발, *각각 실제 심은 위반을 정확히 명명*. **S2(tell 없는 self-invocation)도 잡음 → PR #11 footgun 체크리스트가 정답패턴 없이도 작동함을 검증.**
- **의도한 오탐 유혹 = 0/2 물림** — C1에서 "인터페이스 없다"고 *안* 했고(YAGNI 인정), C2에서 "record는 보일러플레이트 없다"고 *안* 막음(record VO 정당 인정). **하드 케이스에서 정밀도 강함.**
- **판별 = 의도 축 4/4** — 특히 추상화 축에서 과설계(S3)는 막고 무인터페이스 단일구현(C1)엔 *인터페이스를 요구하지 않음*.

### 보너스 품질 신호 (단순 판정 외)
- S1: 코드 임계값(70/45/20)이 *기존 도메인(75/50/25)과 다름* 까지 적발(계약 불일치).
- S3: ground-check로 *해당 타입 참조 0건* 확인 후 과설계 판정(추측 아닌 grounding).
- C4: 보상-실패(flow-C) 누락 + `recall()` 예외가 원본 예외 삼킴을 *관찰*했으나 "비차단 견고성 보강"으로 정확히 판단(과탐 안 함).
- C2: 대상 경로/내용이 실제 repo와 다름을 알아채고도 올바르게 통과.

## ⚠️ 한계 / 결함 (정직)

- **C1 샘플 오염(내 잘못):** "인터페이스 없는 단일구현=깨끗"으로 깔았으나, 그 거리계산이 *기존 `Coordinate.distanceKmTo`를 복붙*한 꼴이라 적대자가 DRY/빈약도메인으로 막았다. → **내가 테스트한 오탐(인터페이스 요구)은 안 했고, 대신 진짜 결함(중복)을 잡았다.** 즉 C1의 block은 *오탐이 아니라 (의도치 않은) 정탐*, 단 "깨끗 샘플"로는 무효. 정밀도 분모는 사실상 C2·C3·C4 = 3개, 그중 오탐 0.
- **n=8 파일럿.** 비율 신뢰구간 넓음.
- **inline이라 적대자가 ground-check 컴파일 못 씀**(일부는 했지만 대상이 디스크에 없어 협력자만 확인) → 이 측정은 *의미판단 단독*. 실제 훅은 컴파일 신호도 받아 더 강함 → **하한 추정.**
- **tell 의존:** S1·S4는 기존 도메인이 대조군(catch 쉬움). 단 S2(tell 없음)도 잡았으니 footgun엔 tell 불요 확인. 추론으로만 풀리는 위반(과설계 S3)도 잡음.

## 결론 — 검증 절반은 *실제로 좋다*

> 생성 절반(주입)은 0이었지만, **검증 절반(적대자)은 recall 4/4 + 하드 오탐 0 + 의도 축 4/4 판별**로 신뢰도가 높게 나왔다. 잘 알려진 위반(DDD·캡슐화)뿐 아니라 *tell 없는 footgun(S2)*과 *추론형 과설계(S3)*까지 잡고, *YAGNI 단일구현·record VO 같은 오탐 유혹*엔 안 물렸다. eval 전체에서 **가장 강한 positive.** 이는 "하네스 값 = 검증(적대자)에 있다"(DRIFT)는 결론을 *숫자로* 뒷받침한다. 단 C1 샘플 오염·n=8·의미판단 단독(하한)이라는 한계 위에서.
